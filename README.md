# **리눅스 명령어**


  > ## 1) top 명령어
  >> ### 1-1 ) 개념
  >>        top 명령어는 현재 OS의 상태를 나타내주는 명령어 입니다. 
  >>        메모리 사용량, CPU 사용량 등을 나타내주며 top를 실행하는 동안에는 주기적인 업데이트로 실시간에 근접한 내용을 보여줍니다. 
  >>        이를 통하여 리눅스를 사용하는 디바이스의 성능이나 리눅스 서버의 성능을 체크할 때 매우 유용합니다.

> ## 3)  jobs 명령어
>> ### 3-1 ) 개념
>>       jobs 명령은 현재 쉘 환경에서 시작된 작업의 상태를 표시합니다. 
>>       JobID 매개변수로 특정 작업이 지정되지 않으면 모든 활성 작업에 대한 상태 정보가 표시됩니다.
>>       작업 종료가 보고되는 경우, 쉘은 현재 쉘 환경에 알려진 작업 목록에서 해당 작업의 프로세스 ID를 제거합니다.
>>       구문은 jobs [  -l | -n | -p ] [ JobID ... ] 로 사용됩니다.
>> ### 3-2) 구성      
 >> |항목|설명|
 >> |------|---|
 >> | job-number |wait, fg, bg, kill 명령과 함께 사용할 프로세스 그룹 번호를 표시함|
 >> | current |+(더하기 부호)는 fg 또는 bg 명령에 대해 디폴트로 사용될 작업을 식별합니다. </br> 이 작업 ID는 또한 %+(퍼센트 부호, 더하기 부호) 또는 %%(이중 퍼센트 부호)를 사용하여 지정될 수 있습니다. </br> -(빼기 부호)는 현재 디폴트 작업이 종료하는 경우 디폴트가 되는 작업을 식별합니다. </br> 이 작업 ID는 %-(퍼센트 부호, 빼기 부호)를 사용하여 지정할 수도 있습니다. </br> 다른 작업의 경우에는 current 필드가 공백 문자입니다. </br> +로는 하나의 작업만 식별될 수 있고 -로도 하나의 작업만 식별될 수 있습니다. </br> 일시 중단된 단일 작업이 있는 경우 현재 작업이 됩니다. </br> 최소 두 개의 일시 중단된 작업이 있는 경우에는 이전 작업 또한 일시 중단됩니다.|
 >> |state| Running :작업이 신호에 의해 일시중단되지 않았으며 종료하지 않았음을 나타냅니다. </br> Done : 작업이 완료되어 종료 상태 0을 리턴했음을 나타냅니다. </br> Done (code) :작업이 정상적으로 완료되었으며 지정된 0이 아닌 종료 상태 코드로 종료했음을 나타냅니다. 이 코드는 10진수로 표시됩니다. </br> Stopped : 작업이 일시 중단되었음을 나타냅니다. </br> Stopped (SIGTSTP) :SIGTSTP 신호가 작업을 일시 중단했음을 나타냅니다. </br> Stopped (SIGSTOP):SIGSTOP 신호가 작업을 일시 중단했음을 나타냅니다. </br> Stopped (SIGTTIN) :SIGTTIN 신호가 작업을 일시 중단했음을 나타냅니다. </br> Stopped (SIGTTOU) : SIGTTOU 신호가 작업을 일시 중단했음을 나타냅니다.|
 >> | command |쉘과 관련된 명령|
